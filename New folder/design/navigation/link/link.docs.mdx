The `Link` component provides a versatile way to handle both internal application navigation and external website redirection. It leverages `react-router-dom` for internal routing, offering a seamless user experience within the application, and standard `<a>` tags for external destinations.

It supports various states including disabled, custom styling, and provides options for opening links in new tabs.

### Basic Internal Navigation

This example demonstrates how to create a basic internal link using the Link component for in-app navigation.

```tsx
<Link href="/dashboard">Go to Dashboard</Link>
<Link href="/settings">Settings</Link>
```

### External and Disabled Links

This example showcases how to create an external link and how to render a link in a disabled state preventing navigation.

```tsx
<Link href="https://cloudhop.com" external target="_blank">CloudHop Website</Link>
<Link href="/premium" disabled>Premium Features</Link>
```

### Link with Custom Styling

This example illustrates how to create a link without the default styles allowing for complete custom styling and behavior.

```tsx
<Link
  href="/profile"
  noStyles
  style={{
    padding: '8px 16px',
    backgroundColor: '#007bff',
    color: 'white',
    borderRadius: '4px',
    textDecoration: 'none',
    transition: 'background-color 0.2s',
  }}
  className="custom-profile-link"
>
  View Profile
</Link>
```

### React Router Hooks

The link component also exposes useful `react-router-dom` hooks for programmatic navigation and route information:
- `useLocation`: Provides the current URL location object.
- `useParams`: Extracts dynamic parameters from the URL.
- `useSearchQuery` (aliased from `useSearchParams`): Manages and parses the URL query string.
- `useNavigate`: Enables programmatic navigation.
