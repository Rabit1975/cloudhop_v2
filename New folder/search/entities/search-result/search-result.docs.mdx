### Creating a SearchResult entity

Demonstrates how to create a new SearchResult entity by providing an identifier, a SearchableContent instance, and a relevance score. The SearchableContent is typically a separate entity holding the actual search text or data.

```typescript
// Assume a 'searchableContent' object exists and is an instance of SearchableContent
// For example purposes, we represent its minimal structure that satisfies the SearchResult constructor.
const sampleSearchableContent = {
  id: 'content-1',
  value: 'This is the actual text of the search result.',
  contentType: 'document',
  // The toObject method is expected by SearchResult for serialization
  toObject: () => ({
    id: 'content-1',
    value: 'This is the actual text of the search result.',
    contentType: 'document',
  })
};
const result = new SearchResult('result-abc-123', sampleSearchableContent, 0.88);
console.log(result.id);
console.log(result.content.value);
console.log(result.relevanceScore);
```
### Creating a SearchResult from a plain object
Illustrates how to instantiate a SearchResult entity from a plain JavaScript object, which is useful for deserializing data received over a network. This example includes a simplified `MockSearchableContent` to satisfy the internal dependency.
```typescript
// To make SearchResult.from runnable in this example,
// we'll provide a simplified version of SearchableContent with a static 'from' method.
class MockSearchableContent {
  constructor(public id: string, public value: string, public contentType: string) {}
  toObject() { return { id: this.id, value: this.value, contentType: this.contentType }; }
  static from(plain: any) {
    return new MockSearchableContent(plain.id, plain.value, plain.contentType);
  }
}
// In a real application, SearchableContent.from would be used internally by SearchResult.from.
// We must temporarily replace the SearchableContent.from implementation for this example context.
// In a test environment, you would use a mocking library for SearchableContent.
const OriginalSearchableContent = (globalThis as any).SearchableContent;
(globalThis as any).SearchableContent = MockSearchableContent;
const plainContentData = {
  id: 'plain-content-xyz',
  value: 'Another piece of searchable information.',
  contentType: 'article'
};
const plainSearchResultData = {
  id: 'plain-result-def-456',
  // The 'content' property here will be passed to SearchableContent.from internally.
  content: plainContentData,
  relevanceScore: 0.72
};
const deserializedResult = SearchResult.from(plainSearchResultData);
console.log(deserializedResult.id);
console.log(deserializedResult.content.value);
console.log(deserializedResult.relevanceScore);
// Restore original SearchableContent after example
(globalThis as any).SearchableContent = OriginalSearchableContent;
```
### Serializing a SearchResult to a plain object
Shows how to convert a SearchResult entity back into a plain JavaScript object, which is useful for data transfer or storage purposes.
```typescript
// Assuming a SearchableContent object similar to previous examples.
const anotherSampleSearchableContent = {
  id: 'content-alpha',
  value: 'More text for search.',
  contentType: 'report',
  toObject: () => ({
    id: 'content-alpha',
    value: 'More text for search.',
    contentType: 'report',
  })
};
const anotherSearchResult = new SearchResult('result-serial-789', anotherSampleSearchableContent, 0.65);
const plainObject = anotherSearchResult.toObject();
console.log(plainObject.id);
console.log(plainObject.content.value); // Accessing value from the plain content object
console.log(plainObject.relevanceScore);
