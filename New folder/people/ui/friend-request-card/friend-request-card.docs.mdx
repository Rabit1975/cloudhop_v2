---
description: A UI component to display a pending friend request.
labels: ['friend-request', 'card', 'social']
---

A `FriendRequestCard` component allows users to view and respond to pending friend requests. It prominently displays the sender's avatar and name, along with 'Accept' and 'Reject' buttons. The sender's name links to their profile page.

### Display a pending friend request

This example shows a basic friend request card with a user avatar and name. The card includes buttons to accept or reject the request.

```tsx
// In a real application, FriendRequest and UserProfile would be fetched from your backend.
// For documentation purposes, we use mock data.
const mockRequest = {
  id: 'req_123',
  fromUserId: 'user_active',
  toUserId: 'user_me',
  status: 'pending',
};
const mockUserProfile = {
  id: 'user_active',
  userId: 'user_active',
  username: 'nebula_walker',
  email: 'walker@nebula.io',
  displayName: 'Nebula Walker',
  profilePicture: 'https://storage.googleapis.com/bit-generated-images/images/image_a_sleek__modern_ui_component_f_0_1770835143864.png',
  bio: 'Exploring the digital frontier.',
  createdAt: '2023-01-01T00:00:00Z',
  presenceStatus: 'online',
  socialConnections: [],
  // @ts-ignore
  toObject: () => ({}),
  firstName: 'Nebula',
  lastName: 'Walker',
  statusMessage: '', // Added to satisfy required property
};
<FriendRequestCard
  request={mockRequest}
  userProfileMock={mockUserProfile}
/>
```
### Handling requests from users without avatars
When a user does not have an avatar image, their initials are displayed within the avatar component instead.
